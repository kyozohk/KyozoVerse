Actual result: Both fields accept and store strings of any length without any limit or error. A 370+ character name and a 500+ character email were successfully imported and persisted to the member record with no warning.
Assignee: Ashok
Bug title: No character length limit on Name or Email fields in Manual Entry — extremely long strings accepted and stored
Category: Functional
Environment: http://pro.kyozo.com — Chrome
Expected result: Fields should enforce a reasonable maximum length (e.g. 100–255 chars for name, 254 chars for email per RFC 5321). Exceeding the limit should show an inline validation error and prevent submission.
Notes: No maxlength attribute is enforced on the Name or Email inputs. This is a systemic gap across the Manual Entry form. Very long strings also cause downstream UI layout failures (see KYPRO-67). Could also present database storage risks depending on column type and size constraints server-side.
