# KYPRO-39

Actual result: All validation error messages render below the Profile Banner upload section, which is below the visible dialog area without scrolling. The Add Member button appears completely non-functional to users. Additionally, required fields (Last Name, Phone) have no asterisk or "required" indicator, so users cannot know these fields are mandatory before attempting submission.
Assignee: Ashok
Bug title: Add Member form – all validation errors hidden below the fold
Category: UX
Environment: Web – http://pro.kyozo.com (QA Test Community)
Expected result: Validation errors should appear immediately adjacent to the relevant field (inline), or at minimum at the top of the form — always visible without scrolling. Required fields should be marked with an asterisk (*) or a "required" label.
Notes: During QA testing this bug caused the Add Member button to appear broken across multiple test cases. High severity because it directly blocks users from creating members without knowing they need to scroll down. Three distinct error messages are affected: phone required, name required, invalid email.
Page/Feature: Audience > Add Member
Priority: High

Steps to reproduce: Scenario A – missing phone:
1. Click Add Member in the Audience header
2. Fill First Name, Last Name, valid Email; leave Phone blank
3. Click Add Member
4. Nothing visible happens → scroll down past Profile Banner
5. Error appears: "Phone number is required."

Scenario B – missing last name:
1. Click Add Member; fill First Name, Email, Phone; leave Last Name blank
2. Click Add Member → scroll down
3. Error appears: "First and last names are required."

Scenario C – invalid email:
1. Click Add Member; fill First Name, Last Name, Phone; enter "notavalidemail" in Email
2. Click Add Member → scroll down
3. Error appears: "A valid email address is required."
