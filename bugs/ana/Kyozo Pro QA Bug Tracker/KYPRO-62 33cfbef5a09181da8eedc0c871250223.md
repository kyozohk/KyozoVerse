# KYPRO-62

Actual result: Tab delimiter is not recognised. Every column (NAME, EMAIL, PHONE, TAGS) shows all four fields of the row concatenated as a multi-line block in the preview. If the user proceeds to import, the corrupted data is saved to the database: each contact's Name, Email, and Tags fields all receive the full concatenated row string (e.g. name = 'Diana Prince mailto:diana@example.com +15550103 vip', email = 'Diana Prince mailto:diana@example.com +15550103 vip', tag = 'Diana Prince mailto:diana@example.com +15550103 vip'). The Phone field is left empty. Results in completely unusable audience records with no warning shown to the user.
