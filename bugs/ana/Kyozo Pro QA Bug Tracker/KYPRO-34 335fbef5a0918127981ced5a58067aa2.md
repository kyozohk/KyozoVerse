# KYPRO-34

Actual result: Clicking Add Member with all fields empty does NOT simply do nothing — it silently creates a blank '–' record in the member queue. Each click on Add Member adds another null entry. These blank records are then included in the Preview count and can be imported, creating completely empty contacts in the system with no name, email, phone, or tags.

[NEW - confirmed 2026-04-08] The same missing validation applies to the inline edit flow in the preview table: clicking ✏️ on any row, clearing all fields, and pressing 💾 also silently saves the empty row with no error. When that empty row is then imported, it creates a ghost 'Unknown User' member in the Audience. Navigating to that member's profile results in a permanently broken page — a loading skeleton that never resolves (URL: /members/imported-XXXXXX-0). The page never loads and the member is inaccessible.
Assignee: Ashok
Bug title: Empty Manual Entry form submission silently creates blank null records with no validation feedback
Category: UX
Environment: http://pro.kyozo.com — Chrome
Expected result: Clicking Add Member with all fields empty should display a clear validation message indicating that at least Name (or Name + Email) must be filled. The form should reject the submission and not create any record. The member queue should remain unchanged.
Notes: Originally assessed as Low severity (no data corrupted, just silent UI failure). Updated after live reproduction: the bug is worse than initially documented — each empty click creates a null '–' record visible in the queue and counted in Preview (N). These null records can propagate into the import. Severity raised to Medium.

[2026-04-08] Further confirmed via inline edit path (see Path B in steps). This path produces an even more severe consequence: a ghost 'Unknown User' is created in the audience and their profile page permanently fails to load (infinite loading skeleton at /members/imported-XXXXXX-0). Severity should be considered for upgrade to High.
